# OUTPUT 项目说明文档

## 1. 文档目的

本文档仅说明当前工程的硬件连接、按键逻辑、LCD 显示、测量注意事项、非阻塞处理思路、调试方法以及故障排查流程，便于后续联调、测试和维护。

当前工程基于 STM32F407、AD7616 并口采集和 GPIO 驱动 LCD，整体实现目标是：在现有板级条件下稳定完成 B0 通道实时采样、时域显示、FFT 频谱分析和调试信息输出。

## 2. 硬件连接方式说明

### 2.1 主控与采集链路概览

- 主控为 STM32F407。
- 模拟采集芯片为 AD7616。
- 显示设备为一块 GPIO 驱动的 LCD。
- 当前工程固定采集 AD7616 的 pair 0，对应 A0 和 B0，其中频谱分析只使用 B0。
- 系统通过定时器中断驱动采样，通过主循环完成 FFT 和显示刷新。

### 2.2 按键与指示灯连接

- 用户按键为 KEY，引脚定义在 [Core/Inc/main.h](Core/Inc/main.h#L60)。
- 红灯为运行状态/过载指示灯，引脚定义在 [Core/Inc/main.h](Core/Inc/main.h#L70)。
- 蓝灯为主循环活动指示灯，引脚定义在 [Core/Inc/main.h](Core/Inc/main.h#L72)。

### 2.3 AD7616 连接方式

当前 AD7616 工作在并口模式，主要控制脚如下：

- RD： [Core/Inc/main.h](Core/Inc/main.h#L74)
- WR： [Core/Inc/main.h](Core/Inc/main.h#L76)
- CHS0： [Core/Inc/main.h](Core/Inc/main.h#L78)
- CS： [Core/Inc/main.h](Core/Inc/main.h#L80)
- CHS2： [Core/Inc/main.h](Core/Inc/main.h#L82)
- CHS1： [Core/Inc/main.h](Core/Inc/main.h#L84)
- CONV： [Core/Inc/main.h](Core/Inc/main.h#L86)
- BUSY： [Core/Inc/main.h](Core/Inc/main.h#L88)
- RST： [Core/Inc/main.h](Core/Inc/main.h#L90)
- RNG1： [Core/Inc/main.h](Core/Inc/main.h#L92)
- RNG0： [Core/Inc/main.h](Core/Inc/main.h#L94)
- SER/PAR： [Core/Inc/main.h](Core/Inc/main.h#L96)

### 2.4 AD7616 数据总线说明

AD7616 的 16 位数据线并不是连续挂在同一组 GPIO 口上，而是分散在多个端口：

- D15 到 D8 位于 GPIOD，定义见 [Core/Inc/main.h](Core/Inc/main.h#L98-L113)
- D7 到 D4 位于 GPIOC，定义见 [Core/Inc/main.h](Core/Inc/main.h#L114-L121)
- D3 到 D0 位于 GPIOA，定义见 [Core/Inc/main.h](Core/Inc/main.h#L122-L129)

这意味着当前板级无法把 AD7616 数据口当作一个连续并行外设总线直接交给 DMA 搬运，因此工程采用逐位重构数据字的方式读取转换结果。

### 2.5 LCD 连接方式

LCD 当前采用 GPIO 模拟时序驱动，关键引脚如下：

- DC： [Core/Inc/main.h](Core/Inc/main.h#L138)
- RES： [Core/Inc/main.h](Core/Inc/main.h#L142)
- SDA： [Core/Inc/main.h](Core/Inc/main.h#L146)
- SCL： [Core/Inc/main.h](Core/Inc/main.h#L150)

此外，显示相关控制脚还包括 BLK、FS、CS、PS、FSY、RST 等，定义分布在 [Core/Inc/main.h](Core/Inc/main.h#L130-L153)。当前 LCD 不是硬件 SPI DMA 刷屏，因此必须通过限流刷新方式避免主循环被长时间占用。

## 3. 各个按钮的功能作用

### 3.1 KEY 按键

当前工程只使用一个用户按键 KEY。

功能如下：

- 短按一次，在主显示页和 Debug 页之间切换。
- 按键只影响显示页面，不影响采样时钟、中断采样、FFT 计算和自动量程逻辑。

### 3.2 当前没有启用的按键功能

为了保证主流程清晰、稳定、可落地，当前版本没有启用以下行为：

- 不通过按键切换 B0/B1。
- 不通过按键切换不同采样通道对。
- 不通过按键触发扫频、校准或阻塞式测试流程。

## 4. LCD 显示内容与逻辑

### 4.1 主显示页内容

主显示页用于正常观测测量结果，主要内容包括：

- 当前运行标识，例如 B0 FFT SCOPE。
- 当前量程状态，例如 5V 或 10V。
- A0、B0 的最新瞬时电压值。
- 当前 FFT 主峰频率。
- 当前累计丢帧数。
- B0 的时域波形。
- B0 的频谱柱状图。

### 4.2 Debug 页内容

Debug 页主要用于运行状态观察和问题定位，当前显示内容包括：

- 实际采样率。
- FFT 主峰频率。
- 丢帧计数。
- AD7616 超时计数。
- 当前帧峰值码值。
- 当前量程状态。
- 当前是否过载。

### 4.3 LCD 刷新逻辑

LCD 不是整屏全量无脑刷新，而是按职责拆分显示过程：

- 文本区优先刷新，用于尽快给出状态反馈。
- 时域波形区按列分片清除和重绘。
- 频谱图区按列分片清除和重绘。
- Debug 页以文本为主，刷新负载相对较低。

### 4.4 页面切换逻辑

- 切换页面时只改变显示状态机，不停止采样。
- 即使 LCD 正在切页，采样和 FFT 仍按既定节拍继续运行。
- 如果主页面绘制负担较高，系统优先保证采样节拍，其次才是显示完整性。

## 5. 测量时的注意事项

### 5.1 输入幅度范围

- 当前系统面向 ±5V 到 ±10V 范围内的输入信号。
- 对于未知幅值信号，建议从较低幅度开始逐步接入，避免一上电就处于重载状态。
- 如果前端信号可能超过 10V 档允许范围，应增加前级保护、分压或限幅电路。

### 5.2 频率范围与 FFT 限制

- 当前采样率为 10 kHz。
- 理论奈奎斯特上限为 5 kHz。
- 工程使用 FFT 做频谱分析时，建议把主要关注频段控制在 4 kHz 以内，以降低混叠和边带误判风险。
- 低频段下，频谱峰值读数可能受窗函数和分辨率影响，必要时应结合时域波形一起判断。

### 5.3 自动量程切换注意事项

- 当输入信号较大时，系统会自动从 5V 档切换到 10V 档。
- 当输入信号较小时，系统会自动从 10V 档切回 5V 档。
- 量程切换后会主动丢弃一个样本，用于避开硬件切换瞬间的不稳定值。
- 如果观察到切换瞬间波形有轻微跳变，这是正常现象；如果长时间异常，则需要检查量程脚和输入幅度。

### 5.4 输入信号质量要求

- 若输入频率接近采样上限，建议在模拟前端增加低通滤波，减少混叠。
- 若输入信号噪声较大或失真明显，FFT 主峰可能不稳定。
- 若输入信号为高阻或长线传输，建议注意接地、屏蔽和驱动能力，避免干扰进入 ADC。

## 6. 避免堵塞的方法

### 6.1 整体策略

当前工程在现有硬件条件下不采用伪 DMA，而采用可真实落地的非阻塞方案：

- 定时器中断定节拍采样。
- 帧缓冲和队列负责采样与处理解耦。
- 主循环只负责 FFT 和显示。
- 显示采用分片刷新而不是全屏阻塞刷新。

### 6.2 采样与处理解耦

- TIM2 负责提供固定采样节拍。
- 每次中断只做必要采样动作：切通道、触发转换、等待 BUSY、读回 A/B 数据、写入帧缓冲。
- 一帧采满后只把帧标记为可处理，不在中断内做 FFT 和绘图。

### 6.3 队列化缓冲策略

- 当前工程使用多帧缓存，主循环从 ready 队列中取出完整帧处理。
- 如果主循环来不及处理，系统宁可丢弃完整帧，也不拖慢下一次采样中断。
- 这种策略的核心目标是保持采样节拍稳定，而不是盲目追求每帧都不丢。

### 6.4 LCD 分片刷新策略

- 每次主循环只刷新少量波形列或频谱列。
- 文本和图形分开处理，减少一次刷新时的连续占用时间。
- Debug 页尽量使用低成本文本显示，便于问题定位时降低刷新负担。

### 6.5 为什么当前不使用 DMA

- AD7616 数据线跨 GPIOA、GPIOC、GPIOD 多个端口，无法直接形成规则 DMA 采样通道。
- LCD 当前采用 GPIO 时序驱动，不是硬件 SPI DMA 刷屏。
- 因此当前硬件架构下，真实可行的方案是中断采样加主循环分片处理，而不是伪装成 DMA 工程。

## 7. 调试方法

### 7.1 上电后的第一步检查

上电后建议先确认三个最基础的问题：

- 系统是否正常启动。
- LCD 是否正常初始化。
- 按键和指示灯是否有反应。

如果这些基础项都不正常，优先检查供电、时钟和 GPIO 初始化，不要一开始就怀疑 FFT 算法。

### 7.2 先看指示灯状态

- 蓝灯周期翻转，表示主循环还在运行。
- 红灯常亮，通常说明当前过载或有需要重点关注的异常状态。

### 7.3 再看 Debug 页核心指标

调试时建议先切到 Debug 页，重点观察以下字段：

- SR：采样率是否接近预期值。
- TO：ADC 超时计数是否持续增长。
- DROP：丢帧数是否持续增长。
- PEAK：峰值码值是否与输入幅度匹配。
- RNG：量程是否能在 5V 和 10V 之间合理切换。
- OVR：是否进入过载状态。

### 7.4 再看主显示页

主显示页主要看两个结果是否同时合理：

- 时域波形是否连续、幅值是否正确、是否削顶。
- FFT 主峰频率是否与输入信号一致、是否稳定。

### 7.5 推荐调试顺序

1. 先输入稳定单频信号，避免一开始就接复杂波形。
2. 先验证 A0、B0 电压读数和时域波形是否大致正确。
3. 再观察 FFT 主峰位置是否正确。
4. 然后逐步提高输入幅值，验证自动量程是否合理。
5. 最后做长时间运行测试，观察 TO、DROP 和页面切换时的稳定性。

### 7.6 示波器建议观测点

若需要定位硬件时序问题，建议优先测量以下引脚：

- AD7616_CONV
- AD7616_BUSY
- AD7616_RD
- AD7616_CS
- AD7616_RNG0 / AD7616_RNG1

重点确认：

- CONV 是否按采样节拍稳定输出。
- BUSY 是否在合理时间内释放。
- RD 与 CS 的读数时序是否正常。
- 量程切换时 RNG0、RNG1 是否确实翻转。

## 8. 常见故障排查与排错方法

### 8.1 屏幕不显示或不刷新

可能原因：

- LCD 电源、背光、复位或控制引脚异常。
- 主循环未正常运行。
- 采样异常导致状态机没有得到有效显示数据。

排查方法：

- 先检查 LCD 供电、背光、RES、DC、SCL、SDA 引脚。
- 观察蓝灯是否周期闪烁。
- 确认页面切换按键是否有响应。
- 如果 Debug 页也没有更新，优先排查初始化阶段而不是 FFT 部分。

### 8.2 采样值明显错误

可能原因：

- AD7616 数据线接错或读数位序错误。
- 输入地线和系统地不共地。
- 量程脚配置与实际输入幅度不匹配。

排查方法：

- 用稳定直流或低频正弦信号做基准测试。
- 对照 A0、B0 实测电压与屏幕显示是否一致。
- 检查 D0 到 D15 的板级连接与引脚映射是否一致。
- 检查 RNG0、RNG1 的实际电平是否符合当前量程档位。

### 8.3 FFT 主峰乱跳或频率不准

可能原因：

- 输入频率过低或过高。
- 输入接近奈奎斯特上限，发生混叠。
- 输入噪声大、波形畸变或前端带宽不够。
- 采样帧中存在量程切换或超时扰动。

排查方法：

- 先确认时域波形稳定，再判断 FFT。
- 将输入频率调到几百 Hz 到 1 kHz 范围做验证。
- 检查 Debug 页 TO、DROP 是否异常增加。
- 若必要，可先降低刷新负担，只观察核心数值变化趋势。

### 8.4 ADC 超时持续增长

可能原因：

- BUSY 始终不释放。
- CONV 触发异常。
- RD、CS 时序错误。
- AD7616 供电、复位或模式选择异常。

排查方法：

- 用示波器同时观察 CONV 和 BUSY。
- 确认 AD7616 已处于并口模式。
- 检查 RST 是否有效复位。
- 检查 BUSY 线是否存在上拉、悬空或连接错误问题。

### 8.5 丢帧数持续增加

可能原因：

- 主循环处理不过来。
- LCD 绘制开销过大。
- 页面刷新频率过高。

排查方法：

- 先切到 Debug 页，观察 DROP 是否继续快速增长。
- 如果主页面增长明显而 Debug 页趋于稳定，说明主要瓶颈在图形刷新。
- 应优先减少显示压力，而不是去破坏采样中断节拍。

### 8.6 量程切换后显示异常

可能原因：

- 量程切换瞬间的过渡样本被误读。
- 输入本身已经超出 10V 档能力。
- 量程控制脚没有真正切换到目标状态。

排查方法：

- 观察 Debug 页的 RNG、PEAK、OVR 是否同步变化。
- 用示波器测量 RNG0、RNG1 引脚。
- 若 10V 档仍长期过载，应优先检查输入幅度和前端保护。

## 9. 使用建议总结

当前工程的核心原则不是追求表面上的功能堆叠，而是保证系统可以在现有板级条件下稳定工作。实际使用中应重点把握以下几点：

- 采样节拍稳定优先于显示完整。
- Debug 页优先用于定位系统级问题。
- 主显示页优先用于观察结果和趋势。
- 超时、丢帧、过载三个指标是最重要的运行健康度参考。
- 若现场出现异常，先查硬件时序，再查显示与算法，不要倒置排查顺序。
